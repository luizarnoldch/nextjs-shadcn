# Shadcn Components Categorization

## Actions & Triggers
- **Button**: Triggering actions like "Save", "Submit", or "Delete".
- **Button Group**: Grouping related buttons (e.g., Bold/Italic/Underline).
- **Toggle**: Switching a single setting on/off (e.g., "Mute").
- **Toggle Group**: Selecting one or more values from a set (e.g., Text Alignment).
- **Command**: Power-user search or command palette ($K).
- **Context Menu**: Actions specific to an element on right-click.
- **Dropdown Menu**: A list of actions or links triggered by a button.
- **Menubar**: Traditional top-level navigation (e.g., File, Edit, View).

## Navigation & Layout
- **Accordion**: Collapsible sections for content like FAQs.
- **Breadcrumb**: Path-based navigation (e.g., Home > Settings > Profile).
- **Navigation Menu**: Complex nav for headers with sub-menus.
- **Pagination**: Splitting large data sets across multiple pages.
- **Sidebar**: Primary app-level navigation or dashboard menus.
- **Tabs**: Switching between related views in the same context.
- **Card**: Grouping related information in a structured container.
- **Carousel**: Cycling through images or featured content.
- **Resizable**: Layouts where users can adjust panel sizes.
- **Scroll Area**: Custom-styled scrollbars for overflows.
- **Separator**: Visually dividing content or menu items.
- **Sheet**: Slide-out panels for filters, settings, or mobile menus.
- **Drawer**: Bottom-anchored sheet, often for mobile actions.
- **Aspect Ratio**: Maintaining proportions for images/videos.
- **Collapsible**: Simple toggle to show/hide specific content.

## Input & Form Components
- **Checkbox**: Binary choices or multi-select in a list.
- **Calendar**: Date selection without a text input.
- **Combobox**: Searchable dropdowns for large lists.
- **Date Picker**: Selecting a specific date via a popover.
- **Input**: Basic text, email, or password entry.
- **Input Group**: Combining inputs with icons or buttons.
- **Input OTP**: Secure code entry for 2FA.
- **Label**: Accessible text labels for form fields.
- **Native Select**: Browser-default select for simple mobile support.
- **Radio Group**: Choosing exactly one option from a visible list.
- **Select**: Choosing one version from a list in a popover.
- **Slider**: Selecting a numeric range value (e.g., Volume).
- **Switch**: Instant-effect toggles for settings.
- **Textarea**: Multi-line text entry for comments/descriptions.
- **Field**: Wrapping form elements with validation and labels.

## Data Display & Information
- **Badge**: Tiny labels for status (e.g., "New", "Error").
- **Chart**: Visualizing complex data sets (Line, Bar, Pie).
- **Data Table**: Advanced grids with sorting, filtering, and pagination.
- **Table**: Standard semantic tables for structured data.
- **Skeleton**: Loading placeholders for content.
- **Progress**: Indicating completion of long-running tasks.
- **Avatar**: Representing users with images or initials.
- **Typography**: Consistent text styling (headings, paragraphs).
- **Kbd**: Visual representation of keyboard shortcuts.
- **Empty**: Placeholder for views with no data to show.

## Feedback & Overlays
- **Alert**: Non-disruptive feedback (e.g., "Changes saved").
- **Alert Dialog**: Critical confirmation that forces a decision.
- **Dialog**: Centered overlays for forms or sub-tasks.
- **Hover Card**: Previewing content on mouse hover.
- **Popover**: Contextual information or small forms on click.
- **Sonner**: Stacked toast notifications (Modern).
- **Toast**: Brief, non-interactive status messages.
- **Tooltip**: Brief descriptions on hover for icons/buttons.
- **Spinner**: Visual indicator for loading states.

## Utilities & Others
- **Direction**: Handling RTL/LTR layout directions.
- **Item**: Generic list or menu item wrapper.
